## 🚀 Pico-Celery v0.1.1: Stability & Safety

This release fixes critical runtime errors when using declarative clients and improves the robustness of the worker execution model.

### 🐛 Bug Fixes

  * **Sync Client Support**: Fixed a `RuntimeError` when using synchronous methods in `@celery` clients. The interceptor was incorrectly defined as `async`, forcing an await on synchronous methods. It is now synchronous, restoring full compatibility.
  * **Worker Loop Safety**: The worker task wrapper now detects if an asyncio event loop is already running in the thread (common in some `solo` or custom pool configurations) and safely offloads the task to avoid crashing the worker.

### 🛡️ Configuration Changes

  * **Fail-Fast Defaults**: Removed the hardcoded `redis://localhost` defaults for the broker and backend.
      * **Impact:** If you were relying on these defaults, your app will now fail to start with a `ConfigurationError`. This is intentional to prevent silent failures in production. You must now explicitly provide `CELERY_BROKER_URL` or configure it via YAML/JSON.

### ⬇️ Upgrading

```bash
pip install pico-celery==0.1.1
```
