# 🚀 Pico-Celery v0.2.0 — Ecosystem Integration

This release introduces **native integration with `pico-stack`**, enabling zero-configuration bootstrapping and aligning the package with the latest safety and lifecycle improvements across the Pico ecosystem.

## ✨ New Features

### 🔌 Pico-Stack Auto-Discovery

Added the standard `entry_point` (`pico_stack.modules`).
When using `pico-stack`, `pico-celery` is now **automatically discovered and loaded**, eliminating the need to manually include `"pico_celery"` in your module list.

## 📦 Dependency Updates

### 🧠 Core Upgrade

Minimum requirement bumped to **`pico-ioc >= 2.1.3`**.

**Why?**
This version includes the improved memory-safety model in the IoC container (notably the removal of LRU eviction inside Scopes), which is crucial for the stability of long-running Celery workers.

## ⬇️ Upgrading

```bash
pip install pico-celery==0.2.0
```
